App connectors let a generated bot work with an external service at runtime. A connection belongs to a workspace, while the selected resource is linked to a project branch.

The current built-in connectors include Google Sheets and Supabase. The marketplace can change as VybeBot adds or disables connectors.

## Connection flow

<Steps>
  <Step title="Connect the workspace account">
    Open **Connectors** in the workspace sidebar, select **App connectors**, and open the connector. Complete its OAuth flow.
  </Step>
  <Step title="Choose a project branch">
    Open the project and select the branch that should use the service.
  </Step>
  <Step title="Link a resource">
    Open the connector from the Vyber add menu. Choose an existing resource or create one when the connector supports provisioning.
  </Step>
  <Step title="Ask the agent to integrate it">
    Tell the agent what the bot should read or write. Mention the connector when useful.
  </Step>
  <Step title="Review and deploy">
    Review generated source, managed environment variables, and any external changes before deployment.
  </Step>
</Steps>

## Google Sheets

A Google Sheets connection can be linked to a spreadsheet on the selected branch. VybeBot's connector tools can read ranges, append rows, update ranges, and add worksheets while building.

The generated bot receives managed runtime configuration that calls the VybeBot connector gateway. This avoids placing your Google OAuth tokens directly in project source.

Use Google Sheets for small operational datasets, queues, reports, or configuration that teammates already manage in a spreadsheet.

## Supabase

A Supabase connection can link an existing project or create a new database project in an authorized organization. VybeBot can prepare the client, manage connection variables, generate TypeScript types, and create SQL migration files.

Database migrations have an explicit review flow:

1. The agent generates a migration under `supabase/migrations`.
2. VybeBot records it as pending.
3. You review the SQL and approve its application.
4. Destructive changes require explicit confirmation.

Applied migrations are external database state. Undoing a source file or restoring a project commit does not roll the database back. Create a new rollback migration instead.

## Security and scope

- Workspace OAuth tokens are stored as protected connection data.
- Branch links control which resource the project uses.
- Managed runtime keys can be revoked or refreshed without exposing the underlying OAuth token.
- Integration activity is recorded for auditing.

<Warning>
  Disconnecting an account or unlinking a resource can break a running deployment. Update the branch and redeploy after changing an integration.
</Warning>
