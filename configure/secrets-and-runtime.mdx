Open the project in the Vyber, then select **Settings → Secrets & Configs**. You need the `manage_settings` permission.

## Platform token

Each platform defines its primary credential:

| Platform | Runtime variable | Where to get it |
| --- | --- | --- |
| Discord | `DISCORD_TOKEN` | [Discord Developer Portal](https://discord.com/developers/applications) |
| Telegram | `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) |
| Custom | `PLATFORM_TOKEN` | The API or platform you are connecting |

Discord projects also receive `DISCORD_CLIENT_ID`, which VybeBot derives from the token.

Saved values are masked. Entering a new token replaces the current value. Redeploy after rotating a token.

## Environment variables

Use environment variables for API keys, service URLs, feature flags, and other runtime configuration. Keys must use uppercase letters, numbers, and underscores and can be up to 64 characters.

```text
SUPPORT_CHANNEL_ID
WEATHER_API_KEY
ENABLE_WEEKLY_REPORT
```

Values are encrypted at rest, are write-only after saving, and are not returned to the browser. You can see that a key exists, update its value, or delete it.

<Warning>
  Do not add secrets to source files. A public Explore project or connected Git repository can expose committed values.
</Warning>

Connector-managed variables may also appear in the environment list. Manage those values through the related connector when possible so VybeBot can rotate or refresh them safely.

## Runtime profile

VybeBot manages the deployment Dockerfile. Choose the profile that matches the generated project:

- **Node.js** for most JavaScript and TypeScript bots.
- **Node.js with canvas** when native image or canvas dependencies are required.
- **Python** for requirements-based Python projects.

Discord and Telegram support Node.js and Python templates. Custom platform projects default to Node.js.

Changing the profile affects the next deployment. It does not rewrite the project source or dependency files.

## Before deployment

Confirm that:

- The platform token is configured.
- Every environment key referenced by the code exists.
- The selected runtime matches `package.json` or `requirements.txt`.
- Secret names in code match the saved keys exactly.
- No secret value appears in a project file or chat message.
