VybeBot turns a plain-language idea into a working bot project. You can ask the Vyber agent to write code, review every file it changes, connect external services, and deploy the result to VybeBot Cloud.

You do not need to start with an existing repository. VybeBot creates a runnable project for the platform you choose. Developers can still edit the generated source, use branches, connect GitHub, and inspect runtime logs.

## What you can build

VybeBot includes templates and platform guidance for:

- **Discord bots** built with `discord.js` or `discord.py`
- **Telegram bots** built with `grammY` or `python-telegram-bot`
- **Custom long-running processes** that connect to another platform or protocol

The available platform list is managed by VybeBot. Your workspace may show a different set if an administrator changes the active platforms.

## The usual workflow

```text
Describe your bot → Review generated code → Add credentials → Test and refine → Deploy → Monitor
```

Your AI work consumes AI credits. Running a deployment consumes cloud credits. Both balances belong to the active workspace. See [Billing and credits](/workspace/billing-and-credits) for the distinction.

<Note>
  VybeBot stores your bot token and other secret values as protected configuration. Never paste secrets into a chat prompt or commit them to project files.
</Note>
